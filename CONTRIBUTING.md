# Contributing

Thank you for helping improve CareerAI.

## Workflow

1. Fork the repository.
2. Create a feature branch from main.
3. Keep each pull request focused on one change area.
4. Update documentation when behavior changes.
5. Open a pull request with clear testing notes.

## Local Validation

- Install dependencies from requirements.txt.
- Start app with python backend/app.py.
- Validate /health and /predict endpoints.
- Verify static pages and assets load correctly.

## Pull Request Checklist

- Code runs locally.
- Docs are updated as needed.
- No credentials or private data were committed.
- Changes are minimal and focused.

## Issues

When reporting bugs, include:

- Expected behavior.
- Actual behavior.
- Steps to reproduce.
- Logs or screenshots.
